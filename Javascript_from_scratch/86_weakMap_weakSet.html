"WeakMap and WeakSet are similar to Map and Set, but they hold weak references to objects.
         If an object is no longer referenced elsewhere in the application, it can be garbage collected automatically, helping prevent memory leaks."

        ->Core Characteristics & Restrictions:
            Because these collections feature weak references, they come with a few strict architectural constraints:
            1) Objects Only: Keys in a WeakMap and values in a WeakSet must strictly be objects (or non-registered symbols). Primitives like strings, numbers, or booleans are not allowed.
            2) Not Iterable: You cannot loop over them using forEach, for...of, or access keys/values using methods like .keys() or .values().
            3) No Size Property: You cannot check how many items are inside because the size depends dynamically on the state of the garbage collector.

        ->In short:
            WeakMap : A WeakMap is a special Map where:
                        Keys must be objects
                        Keys are weakly referenced
                        When no other references exist, the object can be garbage collected automatically

        ->Why Is It Called "Weak"?
            Because it doesn't strongly own the object.
            It says: "I'll use this object if it exists, but I won't keep it alive."

        ->No Iteration
            You cannot do:
                weakMap.keys();
                weakMap.values();
                weakMap.entries();
                for (let item of weakMap)
            All are invalid.
